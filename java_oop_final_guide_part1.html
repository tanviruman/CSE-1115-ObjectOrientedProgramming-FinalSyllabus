Topic 02

Abstract Classes

Core Concept
  • Declared with abstract keyword. Cannot be instantiated directly.
  • May contain abstract methods (no body) and concrete methods (with body).
  • May have instance variables, constructors, and static members.
  • A concrete subclass must override all abstract methods, or must also be declared abstract.
  • Use for: common base with some shared implementation, plus enforced overrides.
java · abstract class template
public abstract class Animal {
    String name;             // instance variable ✓

    Animal(String name) {    // constructor ✓
        this.name = name;
    }

    public void eat() {     // concrete method ✓
        System.out.println(name + " eats");
    }

    public abstract void makeSound(); // abstract method — no body!
}

class Bird extends Animal {
    Bird() { super("Bird"); }

    @Override
    public void makeSound() {     // MUST override
        System.out.println("Chirp");
    }
}

// Animal a = new Animal(); ← COMPILE ERROR: cannot instantiate abstract class
Animal b = new Bird();          // OK — parent reference, child object

⚠ Common Pitfalls
  • Trying to instantiate an abstract class: new AbstractFountainPen("x") → compile error. You can only instantiate concrete subclasses.
  • Abstract method with a body: abstract void foo() { } → compile error. Abstract methods have no body — not even empty braces.
  • Forgetting to override ALL abstract methods: If parent has 2 abstract methods and child only overrides 1, the child must be declared abstract too.
  • Abstract class vs Interface confusion: Abstract classes CAN have constructors, instance variables, and concrete methods. Interfaces (pre-Java 8) cannot.
Topic 03

Interfaces

Core Concept
  • All methods are implicitly public abstract. All fields are implicitly public static final.
  • A class implements an interface. An interface extends another interface.
  • A class can implement multiple interfaces (Java's answer to multiple inheritance).
  • An interface reference can hold any object of any implementing class — polymorphism applies.
  • extends always comes before implements in a class declaration.
java · interface template
interface Flyable {
    String MEDIA = "Sky";           // implicitly public static final
    void fly();                      // implicitly public abstract
    boolean needFuel();
}

interface Floatable {
    void canFloat();
}

// Interface extending two interfaces
interface FlyAndFloat extends Flyable, Floatable { }

// Class with extends + implements (extends BEFORE implements)
class Bird extends Animal implements Flyable, Floatable {
    @Override
    public void fly() { System.out.println("Flying in " + Flyable.MEDIA); }
    @Override
    public boolean needFuel() { return false; }
    @Override
    public void canFloat() { System.out.println("Floats in air"); }
}

// Interface as reference type (polymorphism)
Flyable b = new Bird();   // OK — Bird implements Flyable
b.fly();                   // calls Bird.fly() via dynamic dispatch

Spring 2025 Q3 — AdvancedComputable + Adder class
5 marksSpring 2025
java · solution
// Interface inheriting Computable + adding max()
interface AdvancedComputable extends Computable {
    int max(int a, int b);
}

// Adder extends Processor AND implements AdvancedComputable
// extends MUST come before implements
class Adder extends Processor implements AdvancedComputable {

    public Adder(String id) {
        super(id);          // Processor has String id field
    }

    @Override
    public int compute(int a, int b) {
        return a + b;       // sum of two integers
    }

    @Override
    public int max(int a, int b) {
        return (a > b) ? a : b;  // maximum of two
    }

    @Override
    public void process() {
        System.out.println("Processing Addition");
    }
}
⚠ Common Pitfalls
  • implements before extends: Writing class Bird implements Flyable extends Animal is a compile error. The correct order is extends Animal implements Flyable.
  • Trying to add instance variables to an interface: interface X { int count = 0; } — this is actually public static final int count = 0. You cannot make it an instance variable.
  • Interface method with missing implementations: If two interfaces both declare a method with the same signature, the implementing class only needs to provide it once.
  • Interface inheriting multiple interfaces: Use extends (not implements) with a comma-separated list: interface C extends A, B { }.
Topic 04

Abstract vs Interface

Abstract ClassInterface
Can have instance variablesOnly public static final fields
Can have constructorsNo constructors
Can have concrete methodsAll methods implicitly abstract (pre-Java 8)
A class can extend one abstract classA class can implement many interfaces
Child uses extendsClass uses implements; interface uses extends

IS-A relationship (Tiger IS-A Animal)CAN-DO relationship (Bird CAN-DO fly)
✓ Quick Decision Rule

If classes share implementation code → use Abstract Class. If unrelated classes just need to promise a capability (like Comparable, Runnable, Flyable) → use Interface. An abstract class with ALL abstract methods and no fields is functionally identical to an interface — prefer interface in that case.

Topic 05

try / catch / finally

The 6 Golden Rules of try/catch/finally
  1. If try succeeds: skip catch → run finally → continue after try-catch.
  2. If try throws: skip rest of try → match catch → run finally → continue (if caught).
  3. finally ALWAYS runs — even if there is a return, throw, or break inside try or catch.
  4. If catch throws a NEW exception, finally still runs → new exception propagates up.
  5. Closest matching catch wins. If exception unhandled at inner level, propagates to outer try.
  6. Subclass catch must come before superclass catch — otherwise compile error (unreachable catch).
java · flow template
try {
    // risky code
    riskyMethod();                       // may throw
    System.out.println("try success");  // skipped if exception thrown above
} catch (SpecificException e) {        // handle specific first
    System.out.println("specific: " + e.getMessage());
} catch (Exception e) {               // general catch — must come last
    System.out.println("general");
} finally {
    System.out.println("always runs"); // ALWAYS executes
}
System.out.println("after block");   // runs IF exception was caught

Topic 06 — HIGHEST PRIORITY

Exception Output Tracing

🔴 Exam Alert — 6–9 Marks

Exception output tracing appears in EVERY final paper. Use the method-stack mental model: trace from the innermost method outward. For each exception ask: (1) Is there a matching catch at this level? (2) Does finally run here? (3) Does a new exception get thrown from catch?

Summer 2025 Q2 — Full Nested Exception Trace (8 marks)
8 marksSummer 2025

Call chain: main → checkB → checkA. Trace step by step:

Step 1
println "Main function start", enter outer try, println "Inside outer try of main", enter inner try, call checkB().
Step 2
Inside checkB: println "Inside checkB", println "Inside try of checkB", call checkA().
Step 3
Inside checkA: println "Inside checkA", println "Inside try of checkA", then arr[5] = 7 → ArrayIndexOutOfBoundsException thrown.
Step 4
checkA has no catch for ArrayIndexOutOfBounds, but has catch for NumberFormatException. This does NOT match. So the AIOOBE skips the NumberFormatException catch. Then finally of checkA runs: println "Inside finally of checkA". The AIOOBE propagates to checkB.
Step 5
Back in checkB: AIOOBE matches catch(ArrayIndexOutOfBoundsException e). println "Exception caught inside catch1 of checkB". Then throws a NEW ArithmeticException("Thrown from catch1 of checkB"). ArithmeticException propagates to main's inner try.
Step 6
Back in main's inner try: ArithmeticException matches catch(ArithmeticException e). println "Exception caught inside inner catch of main: Thrown from catch1 of checkB". Then inner finally runs: println "Inside inner finally of main".
Step 7
Inner try-catch-finally complete. No unhandled exception. Outer catch is skipped. println "Main function end".
Final Output
Main function start Inside outer try of main Inside checkB Inside try of checkB Inside checkA Inside try of checkA Inside finally of checkA Exception caught inside catch1 of checkB Exception caught inside inner catch of main: Thrown from catch1 of checkB Inside inner finally of main Main function end
⚠ Tracing Pitfalls
  • When catch throws a new exception: The new exception propagates to the CALLER of the current method. The local finally still runs first, then the new exception travels up. The remaining code after the try-catch-finally block does NOT run.
  • ArrayIndexOutOfBoundsException is a RuntimeException (unchecked). It is NOT caught by catch(NumberFormatException e). Only exact-match or superclass catches work.
  • After an exception is caught and handled (no re-throw), execution resumes normally after the entire try-catch-finally block — not where the exception was thrown.
  • Exception at line X in try block: lines X+1 to the end of try are SKIPPED. The catch runs, then finally.
Topic 07

User-Defined Exceptions

Template — Write Every Time the Same Way
  1. Create a class that extends Exception (or extends RuntimeException).
  2. Add constructors that call super(message).
  3. In the method: throws CustomException in signature + throw new CustomException("message") in body.
  4. In caller: try { ... } catch(CustomException e) { e.getMessage() }.

// Task 1: Custom exception
class InsufficientFundsException extends Exception {
    public InsufficientFundsException(String message) {
        super(message);
    }
}

public class Bank {
    private int balance;

    public Bank(int initialBalance) {
        this.balance = initialBalance;
    }

    // Task 2: withdraw method
    public void withdraw(int amount) throws InsufficientFundsException {
        if (amount > balance) {

⚠ Common Pitfalls
  • Forgetting throws in method signature: Any method that throws a checked exception (extends Exception but not RuntimeException) MUST declare throws CustomException. Missing this = compile error.
  • Extending RuntimeException vs Exception: Subclasses of RuntimeException are unchecked — no need for throws declaration. Subclasses of Exception are checked — must declare. On the exam, always extend Exception unless told otherwise.
  • throw vs throws: throw new Foo() — actually throws it (in method body). throws Foo — declares that the method may throw it (in method signature). Both are needed for checked exceptions.

📌 Catch Order Rule

When catching multiple exceptions, always put subclass catches before superclass catches. If you put catch(Exception e) before catch(IOException e), the compiler gives "unreachable catch block" error because Exception already handles everything.

ArrayList & Collections

Fall 2025 Q3 — Student ArrayList: sort, filter, average, remove
9 marksFall 2025
java · full solution
import java.util.*;

public class Sections {
    public static void main(String[] args) {

        // (a) Create ArrayList
        ArrayList<Student> sectionA = new ArrayList<>();

        // (b) Add students
        sectionA.add(new Student("A", 5,  3.61));
        sectionA.add(new Student("D", 4,  3.82));
        sectionA.add(new Student("K", 20, 3.57));
        sectionA.add(new Student("M", 8,  3.70));
        sectionA.add(new Student("L", 15, 3.45));

        // (c) Sort descending by CGPA
        sectionA.sort(Comparator.comparingDouble((Student s) -> s.cgpa).reversed());
        // OR: Collections.sort(sectionA, (a,b) -> Double.compare(b.cgpa, a.cgpa));

        // (d) Highest, Lowest, Average
        double highest = sectionA.get(0).cgpa;
        double lowest  = sectionA.get(sectionA.size() - 1).cgpa;
        double sum = 0;
        for (Student s : sectionA) sum += s.cgpa;
        double avg = sum / sectionA.size();
        System.out.println("Highest: " + highest);
        System.out.println("Lowest: "  + lowest);
        System.out.println("Average: " + avg);

        // (e) Selected: CGPA >= average
        ArrayList<Student> Selected = new ArrayList<>();
        for (Student s : sectionA) {
            if (s.cgpa >= avg) Selected.add(s);
        }

        // (f) Remove all from sectionA
        sectionA.clear();

        // (g) Print Selected using for-each
        for (Student s : Selected) {
            System.out.println(s.name + " " + s.id + " " + s.cgpa);
        }
    }
}
Spring 2025 Q4 — Product ArrayList: insert at index, update, sort by total price
5 marksSpring 2025
java · solution
import java.util.*;

class ProductList {
    public static void main(String[] args) {

        // i. Declare ArrayList
        ArrayList<Product> list = new ArrayList<>();

        // ii. Insert products
        list.add(new Product("Mango",  5,  20));
        list.add(new Product("Apple",  4,  24));
        list.add(new Product("Litchi", 20, 3));

        // iii. Insert Banana at index 1
        list.add(1, new Product("Banana", 10, 12));

        // iv. Update Apple's unit_price to 30
        for (Product p : list) {
            if (p.name.equals("Apple")) { p.unit_price = 30; break; }
        }

        // v. Sort by total price (amount * unit_price) descending
        list.sort((a, b) ->
            Double.compare(b.amount * b.unit_price, a.amount * a.unit_price));

        // Print to verify
        for (Product p : list)
            System.out.println(p.name + ": total=" + (p.amount * p.unit_price));
    }
}
⚠ Common Pitfalls
  • list.add(index, element) vs list.set(index, element): add(1, e) inserts at index 1, shifting everything right. set(1, e) replaces the element at index 1.
  • Sorting: (a, b) → Double.compare(b.val, a.val) for descending. Swapping a and b reverses the order. For ascending use Double.compare(a.val, b.val).
  • ConcurrentModificationException: Never call list.remove() inside a for-each loop on the same list. Use an Iterator or build a new list with the desired elements.
  • Comparable vs Comparator: Comparable is implemented by the class itself (compareTo). Comparator is external logic passed to sort() as a lambda. Use Comparator for on-the-fly sorting logic.

import java.io.*;

public class FixTransactions {
    public static void main(String[] args) {
        String inputFile  = "src/transactions.txt";
        String outputFile = "src/transactions_updated.txt";

        try (BufferedReader br = new BufferedReader(new FileReader(inputFile));
             BufferedWriter bw = new BufferedWriter(new FileWriter(outputFile))) {

            String line;
            while ((line = br.readLine()) != null) {
                // Split on " : " to separate transaction text from amount
                String[] parts = line.split(" : ");
                String senderReceiver = parts[0];       // "Habib sends Labib"
                int    amount          = Integer.parseInt(parts[1].trim());
                int    corrected       = amount - 100;

                bw.write(senderReceiver + " : " + corrected);
                bw.newLine();
            }
            System.out.println("Done. transactions_updated.txt created.");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
Spring 2025 Q2 — Split student.txt into info.txt and mark.txt
10 marksSpring 2025

Read 701 Rabbi 15 12 → write 701 Rabbi to info.txt, Rabbi 27 to mark.txt.

java · solution
import java.io.*;

public class SplitStudentFile {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(new FileReader("src/student.txt"));
             BufferedWriter infoW = new BufferedWriter(new FileWriter("src/info.txt"));
             BufferedWriter markW = new BufferedWriter(new FileWriter("src/mark.txt"))) {

            String line;
            while ((line = br.readLine()) != null) {
                String[] parts = line.split(" ");   // ["701","Rabbi","15","12"]
                String id    = parts[0];
                String name  = parts[1];
                int    mark1 = Integer.parseInt(parts[2]);
                int    mark2 = Integer.parseInt(parts[3]);

                infoW.write(id + " " + name);  infoW.newLine();
                markW.write(name + " " + (mark1 + mark2)); markW.newLine();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
⚠ Common Pitfalls
  • File path: use "src/filename.txt" not just "filename.txt" — the question says "located inside the src folder."
  • Forgetting bw.newLine(): Without this, all output appears on one line.
  • trim() before parseInt: Lines may have trailing spaces. Always parts[i].trim() before parsing numbers.
  • Try-with-resources auto-closes streams — no need to call br.close() or bw.close() explicitly when using try(...) syntax.
Topic 11

Threads

Core Pattern — Every Thread Question
  1. Create a class implementing Runnable (preferred) or extending Thread.
  2. Put the computation in run(). Store result in an instance field.
  3. In main: create threads, call start(), call join() to wait, then combine results.
Fall 2025 Q5 — Series sum S = 1¹ − 2² + 3³ − ... using 2 threads
8 marksFall 2025
java · solution
import java.util.Scanner;

// Thread 1: odd bases (1, 3, 5, ...) — positive terms
class OddThread implements Runnable {
    private int n;
    public double result = 0;

    public OddThread(int n) { this.n = n; }

    @Override
    public void run() {
        for (int i = 1; i <= n; i += 2) {    // i = 1, 3, 5, ...
            result += Math.pow(i, i);           // +i^i
        }
    }
}

// Thread 2: even bases (2, 4, 6, ...) — negative terms
class EvenThread implements Runnable {
    private int n;
    public double result = 0;

    public EvenThread(int n) { this.n = n; }

    @Override
    public void run() {
        for (int i = 2; i <= n; i += 2) {    // i = 2, 4, 6, ...
            result += Math.pow(i, i);           // subtracted later
        }
    }
}

public class SeriesSum {
    public static void main(String[] args) throws InterruptedException {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();   // must be even

        OddThread odd  = new OddThread(n);
        EvenThread even = new EvenThread(n);

        Thread t1 = new Thread(odd);
        Thread t2 = new Thread(even);

        t1.start();   // starts Thread 1
        t2.start();   // starts Thread 2
        t1.join();    // main waits for t1 to finish
        t2.join();    // main waits for t2 to finish

        double totalSum = odd.result - even.result;   // combine: odd adds, even subtracts
        System.out.println("Total sum S = " + totalSum);
    }
}
Summer 2025 Q6 — n! using 2 threads (1 to n/2 and n/2+1 to n)

java · solution
class FactThread implements Runnable {
    private int start, end;
    public long result = 1;   // multiplicative identity

    public FactThread(int start, int end) {
        this.start = start;
        this.end   = end;
    }

    @Override
    public void run() {
        for (int i = start; i <= end; i++) result *= i;
    }
}

public class FactorialThreads {
    public static void main(String[] args) throws InterruptedException {
        int n = new java.util.Scanner(System.in).nextInt();

        FactThread f1 = new FactThread(1,     n/2);
        FactThread f2 = new FactThread(n/2+1, n);

        Thread t1 = new Thread(f1);
        Thread t2 = new Thread(f2);
        t1.start(); t2.start();
        t1.join();  t2.join();

        long factorial = f1.result * f2.result;
        System.out.println(n + "! = " + factorial);
    }
}
⚠ Common Pitfalls
  • Calling run() instead of start(): t.run() runs the code on the MAIN thread (no concurrency). t.start() creates a new thread and runs run() on it.
  • Forgetting join(): Without t1.join(); t2.join();, main may combine results before threads finish, giving wrong answers.
  • main must declare throws InterruptedException when using join(), or wrap it in a try-catch.
Topic 12

Java Swing / GUI Event Handling

Core Pattern — Event Handling

Every GUI question asks you to write event handling code. The pattern is the same every time:

  1. Use a lambda: button.addActionListener(e -> { ... });
  2. Or implement ActionListener: class implements ActionListener, override actionPerformed(ActionEvent e).
  3. To handle multiple buttons in ONE method: use e.getSource() == buttonName.
Summer 2025 Q3 — Word Count Guessing Game (one method for both buttons)
8 marksSummer 2025

B1 (Submit) stores P1's sentence. B2 (Guess) checks if P2's guess matches word count. ONE method handles both — use e.getSource().

java · event handling solution
// Assume: T1, T2 (JTextFields), B1, B2 (JButtons), L1 (JLabel) already created
// We need ONE method to handle both buttons → implement ActionListener

import java.awt.event.*;
import javax.swing.*;

// Declare at class/field level:
String sentence = "";   // stores Player 1's sentence

// Register SAME handler for both buttons:
ActionListener handler = new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        if (e.getSource() == B1) {
            // B1 (Submit): save Player 1's sentence
            sentence = T1.getText();
            T1.setText("");   // optional: clear after submit
        } else if (e.getSource() == B2) {
            // B2 (Guess): check Player 2's word count guess
            int actualWords = sentence.trim().split("\\s+").length;
            int guess;
            try {
                guess = Integer.parseInt(T2.getText().trim());
            } catch (NumberFormatException ex) {
                L1.setText("Please enter a valid number");
                return;
            }
            if (guess == actualWords) {
                L1.setText("Player 2 wins");
            } else {
                L1.setText("Player 1 wins");
            }
        }
    }
};
B1.addActionListener(handler);
B2.addActionListener(handler);   // SAME handler for both

⚠ Common Pitfalls
  • One method for multiple buttons: The question says "use only one method." Use a single ActionListener registered to both buttons, with e.getSource() == buttonRef inside to distinguish.
  • getText() returns a String — always trim() first to handle accidental spaces entered by the user before parsing with Integer.parseInt().
  • Lambda vs anonymous class: Both are valid. Lambda btn.addActionListener(e -> {...}) is shorter. Anonymous class is more explicit. Either earns full marks.
  • Split on whitespace: sentence.split("\\s+") handles multiple spaces between words. split(" ") would create empty strings for multiple consecutive spaces.
