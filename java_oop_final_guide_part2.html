Topic 02

Abstract Classes

Rules — Abstract Class
  • Declared with abstract keyword. Cannot be instantiated (no new AbstractClass()).
  • Can have constructors, fields, concrete methods, and abstract methods.
  • A subclass must implement ALL abstract methods, or it must also be declared abstract.
  • Subclass calls parent constructor with super(...).

Fall 2024 Q1 — Class D with a(), b(), x(), y(), p() methods
5 marksFall 2024

Write non-abstract class D: a()=u+v, b()=u×v, x()=u%v (remainder), y()=√u, p() calls all and prints results. Use u=4, v=10.

java · solution
class D {
    int u, v;

    void setValues(int u, int v) {
        this.u = u;
        this.v = v;
    }

    int a() {
        System.out.println("a()");
        return u + v;              // sum
    }

    int b() {
        System.out.println("b()");
        return u * v;              // product
    }

    int x() {
        System.out.println("x()");
        return u % v;              // remainder
    }

    double y() {
        System.out.println("y()");
        return Math.sqrt(u);       // square root of u
    }

    void p() {
        System.out.println("Sum: "         + a());   // 4+10=14
        System.out.println("Product: "      + b());   // 4*10=40
        System.out.println("Remainder: "    + x());   // 4%10=4
        System.out.println("Square root: "  + y());   // √4=2.0
    }

    public static void main(String[] args) {
        D d = new D();
        d.setValues(4, 10);
        d.p();
    }
}
Topic 03

Interfaces

Rules — Interface
  • All methods are public abstract by default. All fields are public static final.
  • Interface can extend another interface: interface B extends A.
  • Class implements interface: class C implements A, B (multiple allowed).
  • Class can extend one class AND implement multiple interfaces: class C extends Abs implements I1, I2.
  • Order: extends ALWAYS comes before implements.

Spring 2025 Q3 — AdvancedComputable interface + Adder class
5 marksSpring 2025

AdvancedComputable extends Computable (has compute()) and adds max(). Adder extends Processor AND implements AdvancedComputable. Must implement compute() (sum), max(), and process().

// Given (do NOT modify):
interface Computable {
    int compute(int a, int b);
}
abstract class Processor {
    protected String id;
    public Processor(String id) { this.id = id; }
    public abstract void process();
    public void printId() { System.out.println("Processor ID: " + id); }
}

// WRITE THIS: interface AdvancedComputable
interface AdvancedComputable extends Computable {
    int max(int a, int b);   // self-method
}

// WRITE THIS: class Adder
class Adder extends Processor implements AdvancedComputable {

    public Adder(String id) {
        super(id);           // pass id to Processor constructor
    }

    @Override
    public int compute(int a, int b) {
        return a + b;        // i. sum of two integers
    }

    @Override
    public int max(int a, int b) {
        return (a > b) ? a : b;   // ii. maximum of two integers
    }

    @Override
    public void process() {
        System.out.println("Processing Addition");  // iii.
    }
}

Spring 2024 Q1b — ExceptionHandling class with ArrayIndexOutOfBounds + InputMismatch
5 marksSpring 2024

User enters index then value. Type A = ArrayIndexOutOfBoundsException (invalid index), Type B = InputMismatchException (non-integer value). Always print "Exception handling is amazing" in finally.

java · solution
import java.util.*;
class ExceptionHandling {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter array size: ");
        int n = sc.nextInt();
        int[] myArray = new int[n];

        System.out.print("Enter index: ");
        try {
            int index = sc.nextInt();          // may throw InputMismatchException (Type B)
            System.out.print("Enter value: ");
            try {
                int value = sc.nextInt();      // may throw InputMismatchException (Type B)
                myArray[index] = value;          // may throw ArrayIndexOutOfBoundsException (Type A)
                System.out.println("Value inserted!");
            } catch (ArrayIndexOutOfBoundsException e) {
                System.out.println("Type A Exception occurred");
            } catch (InputMismatchException e) {
                System.out.println("Type B Exception occurred");
            } catch (Exception e) {
                System.out.println("Some other exception occurred!");
            } finally {
                System.out.println("Exception handling is amazing");
            }
        } catch (InputMismatchException e) {
            System.out.println("Type B Exception occurred");
            System.out.println("Exception handling is amazing");
        }
    }
}
⚠ Most Common Tracing Mistakes
  • Forgetting finally when exception propagates: If an exception is NOT caught and propagates, finally still runs before propagating.
  • Assuming code after throw runs: Once an exception is thrown (including by throw new ... in a catch block), ALL remaining code in that block is SKIPPED.
  • Confusing catch order: Catches are checked top-to-bottom. Once one matches, others are skipped. Exception catches ALL — put it last.
  • AIOOBE vs NFE: arr[5]=7 (out of bounds) throws ArrayIndexOutOfBoundsException, NOT NumberFormatException. Only Integer.parseInt("abc") throws NFE.

Custom Exceptions

The 3-Line Template — Memorize This
java · template
// Step 1: Create the exception class (always extends Exception)
class MyException extends Exception {
    public MyException(String message) {
        super(message);   // pass to parent Exception constructor
    }
}

// Step 2: Declare "throws MyException" in the method signature
void someMethod(int value) throws MyException {
    if (condition) {
        throw new MyException("Descriptive message here");
    }
    // normal code continues here if no exception
}

// Step 3: Call the method inside try-catch
try {
    someMethod(value);
} catch (MyException e) {
    System.out.println("Caught: " + e.getMessage());
} finally {
    System.out.println("Always runs");
}

import java.util.*;

public class Sections {
    public static void main(String[] args) {
        // (a) Create ArrayList
        ArrayList<Student> sectionA = new ArrayList<>();

        // (b) Insert students
        sectionA.add(new Student("A",  5,  3.61));
        sectionA.add(new Student("D",  4,  3.82));
        sectionA.add(new Student("K", 20,  3.57));
        sectionA.add(new Student("M",  8,  3.70));
        sectionA.add(new Student("L", 15,  3.45));

        // (c) Sort descending by CGPA
        sectionA.sort((s1, s2) -> Double.compare(s2.cgpa, s1.cgpa));
        // After sort: D(3.82), M(3.70), A(3.61), K(3.57), L(3.45)

        // (d) Highest, lowest, average
        double highest = sectionA.get(0).cgpa;
        double lowest  = sectionA.get(sectionA.size() - 1).cgpa;
        double sum = 0;
        for (Student s : sectionA) sum += s.cgpa;

        }
        // Selected: D(3.82), M(3.70), A(3.61)  [all >= 3.63? No: A=3.61 < 3.63]
        // Selected: D(3.82) and M(3.70)

        // (f) Remove all from sectionA
        sectionA.clear();

Main.java — Comparator: sort by CGPA (desc) and ID (asc)
Your CodeUploaded
java · annotated solution
// StudentCgpaComparator: descending order (highest CGPA first)
class StudentCgpaComparator implements Comparator<Student> {
    public int compare(Student s1, Student s2) {
        if (s1.cgpa > s2.cgpa) return -1;  // s1 comes FIRST (higher CGPA)
        if (s1.cgpa < s2.cgpa) return  1;  // s2 comes first
        return 0;
    }
}
// Equivalent lambda: list.sort((a, b) -> Double.compare(b.cgpa, a.cgpa))

// StudentIdComparator: ascending order (lowest ID first)
class StudentIdComparator implements Comparator<Student> {
    public int compare(Student s1, Student s2) {
        int left  = Integer.parseInt(s1.id);
        int right = Integer.parseInt(s2.id);
        if (left > right) return  1;  // s2 comes first (lower ID)
        if (left < right) return -1;  // s1 comes first
        return 0;
    }
}
// Equivalent lambda: list.sort((a, b) -> Integer.compare(Integer.parseInt(a.id), Integer.parseInt(b.id)))

// In main: sort → write to file → sort again → write again
Collections.sort(studentsList, new StudentCgpaComparator());
// write to students_sorted_by_cgpa.txt ...

Collections.sort(studentsList, new StudentIdComparator());
// write to students_sorted_by_id.txt ...
Lambda Shortcut (Exam-Safe)

For any CGPA descending sort: list.sort((a, b) → Double.compare(b.cgpa, a.cgpa));
For any CGPA ascending sort: list.sort((a, b) → Double.compare(a.cgpa, b.cgpa));
You can always use lambdas instead of Comparator classes in the exam — they're accepted.

Topic 10

Multithreading

Two Ways to Create Threads
MethodHowWhen to Use
implements Runnableclass T implements Runnable { public void run(){...} } → new Thread(t).start()More flexible — class can extend another class too
extends Threadclass T extends Thread { public void run(){...} } → t.start()Simpler syntax, Spring 2024 Q2a style
Never Forget These Two Lines

t.start() — creates a new thread and calls run() on it. (NOT t.run()!)
t.join() — main thread WAITS for t to finish before combining results. Without it, you get wrong answers.

Spring 2024 Q2a — MovieTheater booking using extends Thread
5 marksSpring 2024
java · solution
// Assume MovieTheater class has:  int availableSeats; void bookTickets(int n)
class MovieTheater {
    int availableSeats;
    MovieTheater(int seats) { this.availableSeats = seats; }

    synchronized void bookTickets(int n) {
        if (availableSeats >= n) {
            System.out.println(Thread.currentThread().getName() + " booked " + n + " tickets");
            availableSeats -= n;
        } else {
            System.out.println(Thread.currentThread().getName() + " could not book: insufficient seats");
        }
    }
}

// User thread class extending Thread
class User extends Thread {
    MovieTheater m;
    int NumOfTickets;

    User(MovieTheater m, int NumOfTickets, String name) {
        super(name);               // pass thread name to parent Thread class
        this.m = m;
        this.NumOfTickets = NumOfTickets;
    }

    @Override
    public void run() {
        m.bookTickets(NumOfTickets);  // invoke bookTickets on MovieTheater
    }
}

// Main class (Movie)
class Movie {
    public static void main(String[] args) throws InterruptedException {
        MovieTheater theater = new MovieTheater(15);

        User mina   = new User(theater, 6, "Mina");
        User nabil  = new User(theater, 8, "Nabil");
        User farhan = new User(theater, 4, "Farhan");

        mina.start();
        nabil.start();
        farhan.start();

        mina.join();
        nabil.join();
        farhan.join();

        System.out.println("Mina booked: " + mina.NumOfTickets);
        System.out.println("Nabil booked: " + nabil.NumOfTickets);
        System.out.println("Farhan booked: " + farhan.NumOfTickets);
        System.out.println("Remaining seats: " + theater.availableSeats);
    }
}
Summer 2024 Q4 — Thread series: 1 - 2 + 3 - 4 + 5 - ... - 50

Thread 1: adds odd numbers (1+3+5+...+49). Thread 2: adds even numbers (2+4+6+...+50). Result = T1.result - T2.result.

java · solution
class OddSumThread implements Runnable {
    long result = 0;
    @Override
    public void run() {
        for (int i = 1; i <= 49; i += 2)   // 1, 3, 5, ..., 49
            result += i;
    }
}

class EvenSumThread implements Runnable {
    long result = 0;
    @Override
    public void run() {
        for (int i = 2; i <= 50; i += 2)   // 2, 4, 6, ..., 50
            result += i;
    }
}

public class AlternatingSeries {
    public static void main(String[] args) throws InterruptedException {
        OddSumThread  oddObj  = new OddSumThread();
        EvenSumThread evenObj = new EvenSumThread();

        Thread t1 = new Thread(oddObj);
        Thread t2 = new Thread(evenObj);

        t1.start();   t2.start();
        t1.join();    t2.join();    // wait for both

        // 1 - 2 + 3 - 4 + ... = (odd sum) - (even sum)
        long result = oddObj.result - evenObj.result;
        System.out.println("Result = " + result);
        // Odd sum: 1+3+5+...+49 = 625
        // Even sum: 2+4+6+...+50 = 650
        // Result = 625 - 650 = -25
    }
}
